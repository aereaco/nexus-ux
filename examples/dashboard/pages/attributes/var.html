data-var

Demonstrates the data-var attribute declaring single-variable ephemeral lexical memory footprints directly in the DOM template bypassing heavy global definitions.

Syntax & API Usage

<div data-var-counter="0"></div>

<div class="mt-4 grid grid-cols-1 md:grid-cols-2 gap-8 bg-base-200 p-8 rounded-box border shadow-inner">
    <!-- Lexical Scoped Counter (Only exists inside this isolated DOM tree) -->
    <div data-var-clicks="0" class="flex flex-col items-center justify-center w-full p-6 bg-base-100 rounded border border-base-300 shadow">
        <h3 class="font-bold mb-4">Instance A</h3>
        <div class="stat-value text-primary mb-4" data-text="clicks"></div>
        <button class="btn btn-outline btn-primary" data-on-click="clicks++">Mutate Local Var Space</button>
    </div>
    
    <!-- Totally independent Lexical Boundary mapped to the same variable name structurally -->
    <div data-var-clicks="1000" class="flex flex-col items-center justify-center w-full p-6 bg-base-100 rounded border border-base-300 shadow">
        <h3 class="font-bold mb-4">Instance B (Isolated Clone)</h3>
        <div class="stat-value text-secondary mb-4" data-text="clicks"></div>
        <button class="btn btn-outline btn-secondary" data-on-click="clicks += 50">Mutate Independent Space</button>
    </div>
</div>

Notice neither `clicks` space leaks globally. These namespaces are bound physically to the element they attach to.

Extensibility Notes

Ephemeral mapping inside src/modules/attributes/var.ts operates entirely within local closure trees. Extend it for garbage isolation algorithms ensuring Zero memory leaks dynamically.
