### 2.1 MutationObserver Stabilization (`src/engine/observers/mutation.ts`)
**Goal**: Enable `<select>` option sync via ownership-based reactivity without dedicated observers, and prevent duplicate initialization during drag-and-drop reordering — with **no shortcuts**.

**Changes**:
1. **Cross-Batch Move Detection** (robust):
   - Create a module-level `WeakSet<HTMLElement> movedNodes` to track nodes that have been identified as moved across observer callback batches.
   - In each mutation callback, create a fresh `Set<HTMLElement> addedThisBatch` to collect all nodes added in the current `mutationsList`.
   - After processing all mutations in the batch, schedule a microtask to clear `movedNodes` entries older than the current animation frame using `requestAnimationFrame` or `queueMicrotask` + timestamp check. This prevents unbounded growth while allowing multi-batch move sequences to be recognized.
   - When processing `removedNodes`: if `node.isConnected` **or** `movedNodes.has(node)`, skip cleanup and marker deletion — it's a move. If the node is in `addedThisBatch`, add it to `movedNodes` (this captures same-batch moves).

2. **ChildList Parent Pulse** (propagation):
   - After processing `addedNodes` and `removedNodes` **for each MutationRecord**, call `(mutation.target as NexusEnhancedElement)[RUN_EFFECT_RUNNERS_KEY]?.()`.
   - This pulses the effect runners of the parent container (the target of the childList mutation), ensuring that bindings on elements like `<select>` re-evaluate when their `<option>` children change. Each record pulses individually so multiple independent subtree mutations propagate correctly.

3. **Marker Lifecycle**: Preserve existing logic — nodes with `MARKER_KEY` skip reprocessing; marker deleted only on true removal (not moves).

**Implementation details** (module-level state):

const movedNodeTimers = new Map<HTMLElement, number>(); // node → timestamp

  observerType: 'MutationObserver',
  observe: (el: HTMLElement, context: RuntimeContext) => {
    try {
      let isProcessing = false;
      const observer = new MutationObserver((mutationsList) => {
        if (isProcessing) return;
        isProcessing = true;
        const addedThisBatch = new Set<HTMLElement>();
        const now = performance.now();

        // Purge stale moved node entries (>2 frames old ~32ms)
        for (const [node, ts] of movedNodeTimers) {
          if (now - ts > 32) {
            movedNodeTimers.delete(node);
            movedNodes.delete(node);
          }
        }

        try {
          for (const mutation of mutationsList) {
            if (mutation.type === 'childList') {
              // Process added nodes
              mutation.addedNodes.forEach(node => {
                if (node instanceof HTMLElement) {
                  addedThisBatch.add(node);
                  if ((node as NexusEnhancedElement)[MARKER_KEY]) return;
                  context.processElement(node);
                }
              });

              // Process removed nodes with move detection
              mutation.removedNodes.forEach(node => {
                if (node instanceof HTMLElement) {
                  if (node.isConnected || movedNodes.has(node)) return;
                  const enhanced = node as NexusEnhancedElement;
                  if (enhanced[CLEANUP_FUNCTIONS_KEY]) {
                    enhanced[CLEANUP_FUNCTIONS_KEY]!.forEach(c => c());
                    delete enhanced[CLEANUP_FUNCTIONS_KEY];
                  }
                  delete (enhanced as any)[MARKER_KEY];
                }
              });

              // Pulse parent effect for this subtree mutation
              (mutation.target as NexusEnhancedElement)[RUN_EFFECT_RUNNERS_KEY]?.();
            }
            // ... attributes handling unchanged ...
          }

          // Track moved nodes for cross-batch stability
          addedThisBatch.forEach(node => {
            movedNodes.add(node);
            movedNodeTimers.set(node, performance.now());
          });
        } finally {
          isProcessing = false;
        }
      });

      observer.observe(el, { childList: true, subtree: true, attributes: true });
      return () => observer.disconnect();
    } catch (e) {
      reportError(new Error(`Failed to init MutationObserver: ${e instanceof Error ? e.message : String(e)}`), el);
    }
  },
};

### 2.8 Architectural Violation — Intersect Modifier (`src/modules/modifiers/intersect.ts`)
**Goal**: Eliminate **all** direct `IntersectionObserver` instantiation. Use the centralized engine observer exclusively — no shortcuts.

**Strategy**:
The engine's `intersectionObserver` (in `engine/observers/intersection.ts`) creates a per-element IntersectionObserver when `attachObserver('intersection', el, runtime)` is called. It emits a `ux-intersection` custom event (bubbles: true) with detail `{ entry, element, isIntersecting, intersectionRatio }`. The modifier must:
1. **Delegate observation** by calling `attachObserver('intersection', el, runtime)` — this hands off all observer lifecycle to the engine. The returned cleanup (if any) must be stored.
2. **Listen for `ux-intersection`** on the element and invoke the payload or resolve the promise when the `:intersect` condition is met.
3. **Critically**: The modifier must NOT call `new IntersectionObserver()` directly. All observation goes through the centralized registry.

**Full implementation structure**:
```typescript
import { ModifierModule } from '../../engine/modules.ts';
import { RuntimeContext } from '../../engine/composition.ts';
import { CLEANUP_FUNCTIONS_KEY } from '../../engine/consts.ts';
import { attachObserver } from '../../engine/observers'; // NEW: centralized observer API

export const intersectModifier: ModifierModule = {
  name: 'intersect',
  handle: (payload, el, arg, runtime) => {
    const triggerOnLeave = arg === 'leave';
    const triggerOnce = arg === 'once';

    // 1. Delegate observation to engine's centralized intersection observer
    const observerCleanup = attachObserver('intersection', el, runtime);

    // 2. Predicate for intersection condition
    const shouldTrigger = (detail: { isIntersecting: boolean }) =>
      (!triggerOnLeave && detail.isIntersecting) || (triggerOnLeave && !detail.isIntersecting);

    // 3. Event-wrapper mode: payload is a function
    if (typeof payload === 'function') {
      const handler = (e: Event) => {
        const detail = (e as CustomEvent).detail;
        if (shouldTrigger(detail)) {
          if (triggerOnce) cleanup();
          payload(e, detail);
        }
      };
      el.addEventListener('ux-intersection', handler);
      const cleanupFns: (() => void)[] = [() => el.removeEventListener('ux-intersection', handler)];
      if (observerCleanup) cleanupFns.push(observerCleanup);
      return () => cleanupFns.forEach(fn => fn());
    }

    // 4. Pipeline-interceptor mode: returns Promise that resolves on intersection
    return (evalEl, expr, extras) => {
      return new Promise((resolve) => {
        const handler = (e: Event) => {
          const detail = (e as CustomEvent).detail;
          if (shouldTrigger(detail)) {
            if (triggerOnce) {
              observerCleanup?.();
            } else {
              el.removeEventListener('ux-intersection', handler);
            }
            resolve(runtime.evaluate(evalEl, expr, extras));
          }
        };
        el.addEventListener('ux-intersection', handler);
        // Store handler reference for cleanup (captured in closure)
      });
    };
  }
};
```

**Critical invariants**:
- The modifier must not assume the observer is already attached; `attachObserver` is called by the engine during root initialization. The `ux-intersection` event will fire whenever the element's intersection state changes.
- For `triggerOnce`, use `{ once: true }` listener option or explicitly remove listener after first trigger.
- Cleanup must remove the custom event listener to avoid memory leaks.
